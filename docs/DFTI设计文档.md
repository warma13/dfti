# DFTI 三角洲行动人格测试 — 设计文档

> Delta Force Type Indicator  
> 四维度 × 二极 = 16 种人格类型

---

## 一、维度设计

基于《三角洲行动》核心人格冲突，提炼 4 个二元维度：

| 维度 ID | 左极 (正分) | 右极 (负分) | 核心冲突 |
|---------|-----------|-----------|---------|
| **AD** | **A** 猛攻 (Attack) | **D** 防守 (Defend) | 你倾向进攻还是防守？ |
| **TL** | **T** 团队 (Team) | **L** 独狼 (Lone) | 你依赖团队还是单干？ |
| **CR** | **C** 战斗 (Combat) | **R** 资源 (Resource) | 你追求击杀还是搜刮？ |
| **SI** | **S** 策略 (Strategic) | **I** 直觉 (Instinct) | 你靠计划还是靠本能？ |

每个维度独立计分，最终将 4 个维度组合成一个 **4 字母类型代码**（如 `ATCS`、`DLRI`）。

---

## 二、计分规则

### 2.1 单题计分

每道题有 4 个选项，每个选项携带一个 `scores` 表，格式为 `{ 维度ID = 分值 }`。

- **正分** → 偏向该维度的 **左极**（A / T / C / S）
- **负分** → 偏向该维度的 **右极**（D / L / R / I）
- **0 分** → 该维度不受影响
- 一个选项可以 **同时影响多个维度**（跨维度计分）

分值范围通常为 `-2` 到 `+2`：

| 分值 | 含义 |
|------|------|
| +2 | 强烈偏向左极 |
| +1 | 轻微偏向左极 |
| 0 | 中性 |
| -1 | 轻微偏向右极 |
| -2 | 强烈偏向右极 |

### 2.2 累加计分

所有题目的分数按维度 **线性累加**：

```
维度最终得分 = Σ (该维度上所有已选选项的分值)
```

例如：玩家在 AD 维度上选了 +2、-1、+1 三次计分，则 AD 最终得分 = 2 + (-1) + 1 = **2**。

### 2.3 类型判定

每个维度根据最终得分的 **正负** 决定极性：

```
得分 ≥ 0 → 左极字母 (A / T / C / S)
得分 < 0 → 右极字母 (D / L / R / I)
```

4 个维度的字母按顺序拼接，形成最终类型代码：

```
类型代码 = AD极性 + TL极性 + CR极性 + SI极性
示例：A + T + C + S → "ATCS"
示例：D + L + R + I → "DLRI"
```

## 三、题目设计

> **待设计** — 建议每维度 6 题，共 24 题。题目设计原则见下方。

### 设计原则

1. **场景驱动**：用具体游戏场景代入，不做自我评价式提问
2. **三层覆盖**：每维度的题目覆盖行为层（你会怎么做）、应激层（突发反应）、价值层（你信什么）
3. **四选项梯度**：每题 4 个选项覆盖 强左极(+2) / 弱左极(+1) / 中性(0) / 弱右极(-1) / 强右极(-2)
4. **跨维度计分**：选项可同时影响多个维度，捕捉行为关联性
5. **场景差异化**：同维度各题覆盖不同游戏阶段，避免雷同
6. **玩家黑话**：选项文本使用社区真实表达方式

### 题目数量

| 方案 | 每维度 | 总计 | 准确性 |
|------|--------|------|--------|
| 当前（不足） | 3 题 | 12 题 | 误判率 ~25% |
| **推荐** | **6 题** | **24 题** | 误判率 ~8% |
| 高准确 | 10 题 | 40 题 | 误判率 ~3% |

## 五、计算公式汇总

```
输入: 玩家对所有题目的选项选择 (每题选 1 个选项)
          ↓
步骤1: 初始化 4 个维度分数为 0
        AD = 0, TL = 0, CR = 0, SI = 0
          ↓
步骤2: 遍历每道已答题目的选项
        对选项的 scores 表中每个 { 维度ID, 分值 }:
          维度分数[维度ID] += 分值
          ↓
步骤3: 判定各维度极性
        AD极性 = AD ≥ 0 ? "A" : "D"
        TL极性 = TL ≥ 0 ? "T" : "L"
        CR极性 = CR ≥ 0 ? "C" : "R"
        SI极性 = SI ≥ 0 ? "S" : "I"
          ↓
步骤4: 拼接类型代码
        typeCode = AD极性 + TL极性 + CR极性 + SI极性
          ↓
步骤5: 查表获取人格结果
        result = types[typeCode]
          ↓
输出: { type, title, emoji, description, traits, operator, color }
```

### 关键细节

1. **得分为 0 归入左极**：`≥ 0` 而非 `> 0`，即平局时偏向 A/T/C/S
2. **跨维度计分**：一个选项可同时影响多个维度，提升测试区分度
3. **回退机制**：TestEngine 支持回退到上一题，会自动撤销该题分数
4. **Fallback**：若类型代码未匹配到任何预定义人格（理论上不会发生），返回「未知战士」
